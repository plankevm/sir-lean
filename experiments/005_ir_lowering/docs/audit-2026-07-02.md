This document is the **honesty/quality audit**: what's vacuous, circular, shape-lemma, dead, misplaced, and what's missing versus the original goal.*

Method note: every correctness-class smell (vacuous / circular / tautological-tie / dead-scaffold) was handed to a second agent told to *refute* it. ## 1. TL;DR verdict

The delivered artifact is **real and strong on its own terms** — a general (arbitrary cyclic CFG), axiom-clean, gas-free *world*-conformance theorem for storage arithmetic — but it has **drifted from the stated goal** and carries **substantial vestigial scaffolding**, some of it dressed up (in docstrings) as doing work it does not do.

Three headline-level facts a reader should not miss:

1. **The headline is a conditional over un-built runtime facts.** `lower_conforms_cyclic_assembled` (TieDischarge.lean:4798) *supplies* the per-block ties `hstmtties : ∀ L b, StmtTies …` and `htermties : ∀ L b, TermTies …` (:4813/:4815) as hypotheses, plus `hcall : CallPreservesSelf` (:4804) raw. Nothing in the default build **discharges** these for an actual `lower prog` from the recorded run, and there is **no concrete end-to-end instantiation** of the headline. So it reads: *"IF, for every block, the lowering realises the per-cursor ties (IR gas = machine gas, SSTORE realises, CALL realises) AND self-presence is preserved, THEN the world conforms."* The realisability closure — the step that would make it unconditional — is the main thing missing.

2. **The gas-introspection motivation was silently dropped.** PLAN.md calls a gas-dependent branch "the headline reason the IR needs" gas introspection. The v2/v3 pivot made gas an *exact-value realised oracle*, so the monotonicity law (`Trace.gasMonotone` / `realisedGas_monotone`, RunLog.lean:589) — the declared "ONE law" and the whole intellectual payload — is **proved but load-bearing nowhere** in the conformance cone. The only place the law is actually used (Mono.lean sticky-guard) runs on **hand-written bytecode, not `lower prog`**, and is off-cone.

3. **The final "Plus" sprint is largely vestigial.** The `DriveCorrPlus` strengthening (SelfPresent / gas-align / sload-align fields) and everything built to consume it is threaded **empty and verbatim** and **never cashed into the conclusion**; the `CallPreservesSelf` discharge chain (~260 lines) is built and axiom-checked but **not wired into the headline**. Several docstrings overclaim this as "DISCHARGED."

## 2. What the headline actually says

```
theorem lower_conforms_cyclic_assembled
  (hbase : DriveCorr …)                       -- clean-halt scope premise (honest)
  (hwf   : params.accounts.find? recipient = some acc)
  (hdef  : RunDefinable prog)                 -- benign definability
  (hcall : CallPreservesSelf)                 -- SUPPLIED RAW; discharge chain unwired (§4)
  (hpresent / hjumpPresent / hbranchPresent)  -- static CFG block-presence (dischargeable)
  (hwfl  : WellFormedLowered prog)            -- structural
  (hstmtties : ∀ L b, StmtTies … L b)         -- SUPPLIED per-block ties (gas/sload/sstore/CALL) (§3)
  (htermties : ∀ L b, TermTies … L b)         -- SUPPLIED per-block terminator ties
  (hstkBranch : … ≤ 1024)                     -- static stack-depth bound
  : ∃ O, (∃ last halt, Runs (codeFrame …) last ∧ stepFrame last = .halted halt
            ∧ (observe self …).world = O.world)          -- WORLD channel only
        ∧ RunFrom prog o st₀ T prog.entry O
```

Genuinely good and non-trivial: it **constructs** the IR `RunFrom` (does not assume it), it is **cyclic-general** (uses the runtime `totalGas` measure, not static block-rank — a real result), and the storage-world channel is really tied. But (a) only `.world` is tied, not the halt `.result` (empty-RETURN cut, documented); and (b) the runtime ties are inputs, not outputs.

## 3. The realisability-closure gap (the #1 missing piece)

`StmtTies` (LowerConforms.lean:1353) bundles the per-statement runtime correspondences — including `.gas` pinning the IR's introspected gas to the *exact* machine `GAS` output (:1398), `SstoreRealises`, and `CallRealises`. In **all** headline forms these are supplied. To get the theorem we actually want, we need:

> **`hstmtties`/`htermties` BUILT for `lower prog` from `runWithLog … = some log`** — i.e. discharge the per-cursor ties (esp. `ob = machine-gas` at :1398 and the `CallRealises` seam) *positionally* from the recorder's aligned logs. This is exactly the S3 trace↔recorder bridge that the `DriveCorrPlus` note (TieDischarge.lean:3852) leaves supplied.

Until that lands, plus at least one **concrete instantiation** of the general headline on a real `lower prog`, non-vacuity is only argued piecewise and the flagship is a conditional.

## 10. On the sorry-vs-vacuity policy

The audit vindicates the instinct behind the reconsideration. There are **no sorries**, yet §3-4 show the danger the no-sorry mandate was meant to prevent has appeared in a different form: green theorems that *look* like they discharge obligations but thread them empty (the S3 channel), or prove them in an unwired island (CallPreservesSelf), backed by docstrings that overstate ("DISCHARGED"). A well-placed `sorry` at the realisability-closure boundary (§3) would have been *more* honest than the current supplied `StmtTies`/`TermTies` hypotheses, because it would flag the gap instead of burying it in a hypothesis list. **Recommendation:** the standard should be "no *unacknowledged* gaps," enforced by (a) a docstring on every headline stating precisely what is supplied vs discharged, and (b) treating a supplied hypothesis that *could* be discharged as a `sorry`-equivalent debt, tracked in a backlog — not as a finished result.
