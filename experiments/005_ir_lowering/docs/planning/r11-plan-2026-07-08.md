# R11 dispatch plan

Date: 2026-07-08. Baseline: post-P8/P9 source in `experiments/005_ir_lowering`.
This is a proof plan for R11, not a refactor plan. Do not implement R11 proof
obligations in this planning workflow except for tiny documentation corrections that keep this
plan accurate.

## Current target

The R11 public targets are the three WIP theorems in
`LirLean/V2/Realisability/RealisabilitySpec.lean`:

```lean
theorem lower_conforms {prog : Program} {params : CallParams} {log : RunLog}
    {acc : Account}
    (hcode : params.codeSource = .Code (lower prog))
    (hmod : params.canModifyState = true)
    (hself : params.accounts.find? params.recipient = some acc)
    (hgas : GasConstants.Gjumpdest ≤ params.gas.toNat)
    (hwf : IRWellFormed prog)
    (hcodeFits : codeFits prog)
    (hstk : stackFits prog)
    (hrun : runWithLog params (seedFuel params.gas) = some log)
    (hclean : log.clean)
    (hseams : PrecompileAssumptions prog params) :
    ∃ O : Observable,
      RunFrom prog (entryState params) (realisedGas log)
        (realisedCall log params.recipient) (realisedCreate log params.recipient) prog.entry O
      ∧ Conforms params.recipient log O
```

```lean
theorem lower_conforms_exact ... :
    ∃ O : Observable,
      RunFromAll prog (entryState params) (realisedGas log)
        (realisedCall log params.recipient) (realisedCreate log params.recipient) prog.entry O
      ∧ Conforms params.recipient log O
```

```lean
theorem lower_conforms_gasfree ...
    (hng : NoGasReads prog) ... :
    ∃ O : Observable,
      RunFrom prog (entryState params) (realisedGas log)
        (realisedCall log params.recipient) (realisedCreate log params.recipient) prog.entry O
      ∧ Conforms params.recipient log O
```

The companion proof obligation is:

```lean
theorem realisedGas_nil_of_noGasReads {prog : Program} {params : CallParams} {log : RunLog}
    (hcode : params.codeSource = .Code (lower prog))
    (hng : NoGasReads prog)
    (hwl : WellLowered prog)
    (hrun : runWithLog params (seedFuel params.gas) = some log) :
    realisedGas log = []
```

The public static surface is already the post-P9 shape:

```lean
IRWellFormed prog
codeFits prog
stackFits prog
```

`WellLowered prog` and `WellFormedLowered prog` are internal adapters only. The public theorems
must continue rebuilding `WellLowered` by `wellLowered_of_IRWellFormed hwf hcodeFits hstk`.

## Wiring decisions before proof work

Settle these first. They are small statement/wiring decisions, not broad cleanup.

1. `hsize` for the producer must come from `codeFits`.

   `Producer.runFrom_of_driveCorrLog` currently asks for:

   ```lean
   (hsize : (Lir.flatBytes prog).length ≤ 2 ^ 32)
   ```

   The public theorem already has `hcodeFits : codeFits prog`, and `codeFits` is
   `(flatBytes prog).length < 2 ^ 32`. At the call site, derive `hsize` by
   `Nat.le_of_lt hcodeFits`. Do not expose a new size premise, and do not add a size field to the
   public theorem surface.

2. `CreateResolves` needs an honest reachable-frame seam.

   `Producer.runFrom_of_driveCorrLog` used to ask for:

   ```lean
   (hcreate : ∀ fr', ReachableFrom params fr' → CreateResolves fr')
   ```

   `PrecompileAssumptions` now carries this field, so the public theorem surface remains one
   seam bundle:

   ```lean
   createResolves : ∀ fr', ReachableFrom params fr' → CreateResolves fr'
   ```

   Do not re-expose this as a separate public premise.

3. Exact consumption must be produced directly.

   `lower_conforms_exact` needs `RunFromAll`, not `RunFrom`. Do not derive it afterward from
   `runFromLeft_exists`; that only establishes some leftover suffixes. Plan an exact producer
   sibling, or strengthen the coupled recursion so it returns exact leftovers at the same time as
   the plain run.

4. CREATE is now part of the public R11 statement, not a post-R11 nicety.

   Current source has `Stmt.create`, `CreateStream`, and all three flagship conclusions use
   `realisedCreate log params.recipient`. The current producer skeleton still only threads
   gas/sload/call suffixes: `RecorderCoupled` has no create suffix, `StreamsAligned` pins
   `D = createStreamOf log.creates self` at every boundary, and `StmtTies'` has no create arm.
   That is sufficient for the pre-Step-8 CALL-first skeleton, but not for the public theorem over
   arbitrary `IRWellFormed` programs. Before any chunk claims the plain flagship is closed, the
   plan must either land the CREATE coupled channel or explicitly stop short of R11. The intended
   R11 route is to land it: add the create suffix/prefix twin, a `StmtTies'` create arm, the
   recorded-create realisation lemma, and `simStmt_coupled_create`.

that two adjacent chunks are mechanically inseparable. Every chunk must preserve the gates at the
end of this document.

## Proof obligations versus already-handled cleanup

Proof obligations in scope for R11:

- R6 reachable boundary geometry and `boundaryWalk_of_wl`.
- R3 `callRealises_of_recorded`.
- The CREATE coupled channel: create suffix/prefix in the coupling, create ties, recorded-create
  realisation, and `simStmt_coupled_create`.
- R10a `stmtTies'_of_runWithLog` if it remains open when block-walk work starts.
- Coupled producer internals in `Producer.lean`: gas/sload/call/create arms, block walk, drive
  recursion, top-level producer, and exact variant.
- `realisedGas_nil_of_noGasReads`.
- The three R11 blocker shells in `RealisabilitySpec.lean`.

Already-handled cleanup that must not be reopened as R11:

- Public static surface is already `IRWellFormed`, `codeFits`, `stackFits`.
- `WellLowered` and `WellFormedLowered` are already internal adapters.
- Legacy fuel/materialisation residue is already deleted.
- `NoSlotSource`, rank/fuel, `MatFueled`, and acyclic adapters are not live R11 premises.
- The old conditional/cyclic `lower_conforms_cyclic'` path is not the R11 proof route.

## Green gates

Every proof chunk must leave these true:

```sh
cd experiments/005_ir_lowering
lake build
lake build WIP
rg -n "\bsorry\b" LirLean/V2/Realisability
```

Required interpretation:

- Default `lake build` remains green and sorry-free for the default `LirLean` import cone.
- `lake build WIP` remains green.
- The `rg` output monotonically removes only the targeted tracked debts; new WIP sorries are not
  acceptable unless the dispatch explicitly authorizes a newly discovered named obligation and the
  plan is updated.
- No public theorem premise may mention `WellLowered`, `WellFormedLowered`, legacy
  fuel/materialisation machinery, single-call restrictions, call-count restrictions, supplied
  per-block ties, or a new standalone size premise.

## Risks

- `CreateResolves` seam shape is a real blocker. Without it, the producer's existential cannot be
  assembled honestly.
- R6 may require small default-target engine geometry lemmas. Keep them narrow and proof-facing.
- R3 CALL Piece B may expose missing `resumeAfterCall` computation lemmas.
- CREATE is already in the public theorem surface. A producer that leaves `D` pinned to the whole
  `log.creates` at every boundary cannot close R11 for create-containing programs.
- `stmtTies'_of_runWithLog` is still a derived-ties gap; the producer must not compensate by asking
  users for ties.
- Exact consumption can easily become a false after-the-fact strengthening. It must be produced by
  suffix-tracking recursion over gas, call, and create streams.
- Gasfree still needs the coupled producer for non-gas effects; treating `NoGasReads` as a shortcut
  around CALL/SLOAD/SSTORE would recreate the old supplied-tie problem.
