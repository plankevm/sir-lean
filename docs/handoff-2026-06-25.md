## The target interface (Eduardo's model)

```lean
structure EVMSpec where
  State   : Type                          -- the impl's own state representation
  inject  : Input → State                 -- build own state from shared inputs
  interp  : Bytecode → State → State      -- run bytecode on own state → own state
  observe : State → Observables           -- pipe own state into shared observables

def Refines (S I : EVMSpec) : Prop :=     -- name still open: Refines / ConformsTo / Implements
  ∀ (code : Bytecode) (i : Input),
    I.observe (I.interp code (I.inject i)) = S.observe (S.interp code (S.inject i))
```

Each engine is an instance (own `State`/`interp`/`observe`); same `code`+`Input` thread through both.
**Open modeling choice:** EVM input-state ≠ result-state (`messageCall : CallParams → Except _ CallResult`).
Either use one rich `State` covering both, or split `interp : Bytecode → State → Result`. To be confirmed.
